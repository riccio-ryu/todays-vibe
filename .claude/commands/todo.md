TODO.md를 관리해줘.

사용자 입력: $ARGUMENTS

## 동작 규칙

**1단계 — TODO.md 읽기**
`TODO.md` 전체를 읽는다.

**2단계 — 사용자 입력 의도 파악**

입력이 없거나 "확인", "목록", "뭐 있어" 등이면 → **조회**: 현재 대기 항목을 요약해서 답한다. 파일은 수정하지 않는다.

입력이 작업 내용 설명이면 → **추가**: 아래 추가 형식으로 `## 🔜 다음 작업` 섹션 맨 아래에 새 항목을 추가한다.

입력에 "완료", "삭제", "지워", "done" 등이 포함되면 → **완료 처리**: 해당 항목을 `## 🔜 다음 작업`에서 제거하고 `## ✅ 완료된 항목` 테이블에 오늘 날짜(`$CURRENT_DATE`)와 함께 한 줄로 추가한다.

입력에 "수정", "바꿔", "변경" 등이 포함되면 → **수정**: 해당 항목의 내용을 덮어쓴다.

**3단계 — TODO.md 업데이트**
파악한 의도에 따라 파일을 수정한다. 조회일 때는 수정하지 않는다.

---

## 추가 형식

번호는 기존 항목 다음 번호로 자동 부여한다.

```
### N. 제목 (한 줄 요약)
- **내용**: 무엇을 만드는지 간단히
- **목적**: 왜 필요한지 (사용자가 말한 이유)
- **구현 힌트**: 관련 파일, 라우트, 컴포넌트 등 (알 수 있는 경우만)
```

---

## 완료 처리 형식

`## ✅ 완료된 항목` 테이블에 아래 형식으로 한 줄 추가한다:

```
| YYYY-MM-DD | 항목 제목 요약 |
```

---

## 작성 원칙

- 사용자가 말한 내용을 미래의 Claude가 맥락 없이 읽어도 이해할 수 있게 구체적으로 쓴다
- 구현 힌트는 현재 코드베이스 기준으로 관련 파일/패턴을 짧게 적는다
- 불필요한 말 붙이지 말고 파일 수정 후 "추가했습니다 / 완료 처리했습니다 / 수정했습니다" 한 줄로 마친다
